¿Qué es esta plataforma?

Esta es la plataforma web oficial de Universitario Deportes, diseñada para gestionar membresías de socios, proporcionar información del club y ofrecer beneficios exclusivos a los miembros.

Objetivos Principales:

  • Gestión de Socios: Registro, autenticación y administración de cuentas de socios.
  • Sistema de Membresías: Tres niveles (Básica S/120, Premium, VIP) con beneficios escalonados.
  • Panel de Usuario: Dashboard personalizado con información de membresía y acceso rápido.
  • Panel de Administración: Gestión completa de usuarios y permisos (solo para admins).
  • Información Institucional: Historia del club, noticias, beneficios y más.

Estructura de Páginas

Páginas Públicas (sin login):

  • index.html: Página principal con hero banner, información de membresías y llamadas a la acción.
  • info/club.html: Historia del club, títulos, estadio, jugadores históricos.
  • info/noticias.html: Últimas noticias y actualizaciones del club.
  • info/beneficios.html: Lista completa de beneficios para socios.
  • auth/login.html: Formulario de inicio de sesión con Firebase Authentication.
  • auth/registro.html: Formulario de registro de nuevos socios.

Páginas de Usuario (requieren login):

  • user/dashboard.html: Panel principal con resumen de membresía, estado y opciones de upgrade.
  • user/mis-datos.html: Información personal del socio, edición de datos, gestión de perfil.
  • user/test.html: ESTA PÁGINA - Guía completa del sistema y documentación.

Páginas de Administración (solo admins):

  • admin/panel-admin.html: Panel de control para gestionar todos los usuarios, permisos y configuraciones.

Sistema de Autenticación

Firebase Authentication:

Utilizamos Firebase Authentication para gestionar el registro y login de usuarios de forma segura.

  • Registro: Nuevo usuario → crea cuenta con email/contraseña → Firebase genera UID único.
  • Login: Usuario ingresa credenciales → Firebase valida → crea sesión persistente.
  • Persistencia: La sesión se guarda en localStorage (Firebase LOCAL persistence) + sessionStorage para acceso rápido.
  • Logout: Cierra sesión en Firebase + limpia sessionStorage + redirige a página principal.

¿Cómo funciona el Header Dinámico?

Cuando NO hay sesión: El header muestra botones "Iniciar Sesión" y "Registrar".

Cuando HAY sesión: El header muestra el nombre del usuario + menú desplegable con opciones:
• Dashboard
• Mis Datos
• TEST (esta página)
• Panel Admin (solo si eres admin)
• Cerrar Sesión

Sistema de Navegación

Resolución de Rutas:

El sistema detecta automáticamente en qué carpeta estás y construye rutas relativas correctas. Esto permite que funcione con el protocolo file:/// sin servidor.

// Ejemplo de lógica en script.js:
if (pathname.endsWith('index.html')) {
  userPath = 'user/';
} else if (pathname.includes('/user/')) {
  userPath = '../user/';
}
  • Desde raíz (index.html): Links usan user/, auth/, info/
  • Desde subcarpetas: Links usan ../user/, ../auth/, ../index.html
  • Header dinámico: Se adapta automáticamente según la ubicación actual.

Importante:

Siempre usa rutas relativas. NO uses rutas absolutas como /user/dashboard.html porque fallan con file:///

Flujo de Usuario Típico

1. Usuario Nuevo:

  • Visita index.html → Ve información de membresías
  • Click en "Registrar" → Va a auth/registro.html
  • Completa formulario → Firebase crea cuenta + Firestore guarda datos
  • Redirigido a index.html con sesión activa
  • Header muestra su nombre + menú desplegable

2. Usuario Registrado:

  • Visita index.html → Click en "Iniciar Sesión"
  • Va a auth/login.html → Ingresa credenciales
  • Firebase valida → Sesión creada → Redirige a index.html
  • Header muestra menú de usuario
  • Puede navegar a Dashboard, Mis Datos, TEST

3. Administrador:

  • Login como usuario normal
  • Si is_admin: true en Firestore → Ve opción "Panel Admin" en menú
  • Accede a admin/panel-admin.html
  • Puede gestionar todos los usuarios, editar permisos, ver estadísticas

Firebase Configuration

Proyecto Firebase:

  • Project ID: universitario-deportes
  • Authentication: Email/Password habilitado
  • Firestore Database: Colección users con datos de socios
  • Persistence: firebase.auth.Auth.Persistence.LOCAL

Debugging y Solución de Problemas

Problemas Comunes:

  • El header no muestra usuario: Abre DevTools Console → Verifica errores de Firebase → Limpia caché.
  • Links no funcionan: Revisa la consola para errores de rutas → Asegúrate de usar rutas relativas.
  • Sesión no persiste: Verifica que Firebase persistence está en LOCAL → Revisa sessionStorage.
  • Botones invisibles: Ya resuelto → Cambiamos de blanco sobre blanco a rojo con texto blanco.
  • Flash de botones en carga: Ya resuelto → Usamos opacity: 0 inicial + clase .loaded.

Comandos Útiles:

// Ver usuario actual en consola:
firebase.auth().currentUser

// Ver sessionStorage:
sessionStorage.getItem('currentUser')

// Limpiar caché y recargar:
Ctrl + Shift + Delete → Limpiar todo → F5
